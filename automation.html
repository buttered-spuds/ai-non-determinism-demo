Getting started

To automate interactions with this demo, point Playwright at the page URL, interact with the simulator controls, and assert on the shape of the output rather than its exact text. Because the responses are non-deterministic by design, your assertions should check structural properties — word count, keyword presence, semantic meaning — rather than an exact string match.

  1. Install Playwright for your language of choice (see tabs below).
  2. Navigate to the demo URL using page.goto().
  3. Click Run Prompt or Run 10 Times to trigger responses.
  4. Assert on output cards using semantic or constraint-based checks.

Navigate to the demo

Open the simulator page and click Run Prompt to trigger a response. Replace the URL with your deployed GitHub Pages address. 

import { test, expect } from '@playwright/test';

test('navigate to demo and run prompt', async ({ page }) => {
  // Navigate to the demo page
  await page.goto('https://buttered-spuds.github.io/ai-non-determinism-demo/');

  // Click the Run Prompt button
  await page.getByRole('button', { name: 'Run Prompt' }).click();

  // Wait for a response card to appear
  await expect(page.locator('.output-card')).toBeVisible();
});

Write AI-aware assertions

Instead of asserting on exact text, check structural properties of the output — length, keyword presence, and the absence of error states. 

import { test, expect } from '@playwright/test';

test('output card contains meaningful content', async ({ page }) => {
  await page.goto('https://buttered-spuds.github.io/ai-non-determinism-demo/');
  await page.getByRole('button', { name: 'Run Prompt' }).click();

  const card = page.locator('.output-card').first();
  const text = await card.textContent() ?? '';

  // Constraint-based assertions — no exact match needed
  expect(text.trim().length).toBeGreaterThan(10);
  expect(text.split(' ').length).toBeLessThan(80);

  // Keyword presence (case-insensitive)
  const keywords = ['testing', 'bugs', 'quality', 'automation', 'software'];
  const hasKeyword = keywords.some(kw => text.toLowerCase().includes(kw));
  expect(hasKeyword).toBe(true);
});

Assert on statistical distributions

Click Run 10 Times and assert that a proportion of cards pass your quality bar. This mirrors real-world AI test strategies where you run many samples and accept a pass rate threshold. 

import { test, expect } from '@playwright/test';

test('at least 80 % of 10 responses contain relevant keywords', async ({ page }) => {
  await page.goto('https://buttered-spuds.github.io/ai-non-determinism-demo/');
  await page.getByRole('button', { name: 'Run 10 Times' }).click();

  // Wait until all 10 cards are rendered
  await expect(page.locator('.output-card')).toHaveCount(10);

  const cards  = page.locator('.output-card');
  const count  = await cards.count();
  const keywords = ['testing', 'bugs', 'quality', 'automation'];

  let passing = 0;
  for (let i = 0; i < count; i++) {
    const text = (await cards.nth(i).textContent() ?? '').toLowerCase();
    if (keywords.some(kw => text.includes(kw))) passing++;
  }

  // Statistical assertion — expect ≥ 80 % pass rate
  expect(passing / count).toBeGreaterThanOrEqual(0.8);
});

Tips for testing AI-powered UIs

  • ▸ Never use exact-text assertions. AI outputs vary by design — use toContainText with a keyword, or extract and process the text in your test code.
  • ▸ Assert on structure, not content. Check that a response card appeared, that it is non-empty, and that it doesn't contain error states.
  • ▸ Use statistical thresholds. Run the prompt multiple times and assert that the pass rate meets your quality bar (e.g. ≥ 80 %).
  • ▸ Separate deterministic from non-deterministic checks. UI structure (buttons, layout, labels) can be asserted exactly; AI content cannot.
